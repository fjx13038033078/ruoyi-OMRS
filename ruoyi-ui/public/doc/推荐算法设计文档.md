# 智能衣柜推荐系统 - 推荐算法设计文档

## 一、系统概述

智能衣柜推荐系统采用多种推荐算法相结合的方式，为用户提供个性化的衣物推荐服务。系统支持以下六种推荐策略：

| 推荐类型 | 算法类型 | 主要特点 |
|---------|---------|---------|
| 天气推荐 | 基于规则 | 根据实时天气数据推荐适合的衣物 |
| 场景推荐 | 基于内容 | 根据用户场景偏好推荐衣物 |
| 风格推荐 | 基于内容 | 根据用户风格偏好推荐衣物 |
| 季节推荐 | 基于规则 | 根据当前季节推荐当季衣物 |
| 协同过滤 | 协同过滤 | 基于相似用户行为数据推荐 |
| 混合推荐 | 混合策略 | 融合多种算法的综合推荐 |

---

## 二、算法详细设计

### 2.1 基于天气的推荐算法

#### 2.1.1 算法原理

天气推荐算法通过获取用户所在城市的实时天气数据，根据温度、天气状况、湿度等因素，推荐适合当前天气的衣物。

#### 2.1.2 核心逻辑

```
输入：用户ID, 城市名称（可选）
输出：推荐衣物列表, 天气信息, 穿搭建议

流程：
1. 获取城市名称（优先用户输入 > 用户设置 > 默认城市）
2. 调用天气API获取实时天气数据
3. 根据体感温度确定推荐季节
4. 根据温度确定推荐衣物类别及权重
5. 根据天气状况调整推荐策略
6. 查询符合条件的衣物并返回
```

#### 2.1.3 温度-季节映射规则

| 体感温度 | 推荐季节 | 主要推荐类别 |
|---------|---------|-------------|
| ≥28°C | 夏季 | 短袖、短裤、裙装、凉鞋 |
| 20-28°C | 春/秋季 | 长袖、薄外套、休闲裤 |
| 10-20°C | 春/秋季 | 长袖、外套、长裤 |
| 0-10°C | 秋/冬季 | 厚外套、毛衣、保暖裤 |
| <0°C | 冬季 | 羽绒服、保暖内衣、围巾手套 |

#### 2.1.4 天气状况调整策略

```java
// 雨天：增加外套权重，推荐防水鞋
if (condition.contains("雨")) {
    categories.put("outerwear", weight + 0.3);
    categories.put("shoes", 0.9); // 防水鞋
}

// 雪天：全面保暖
if (condition.contains("雪")) {
    categories.put("outerwear", 1.0);
    categories.put("accessories", 1.0); // 围巾手套
    categories.put("shoes", 1.0);       // 防滑保暖鞋
}
```

#### 2.1.5 材质推荐规则

| 温度范围 | 推荐材质 |
|---------|---------|
| ≥28°C | 棉、麻、雪纺（透气吸汗） |
| 20-28°C | 棉、涤纶、混纺 |
| 10-20°C | 棉、涤纶、牛仔 |
| <10°C | 羊毛、羽绒、羊绒、抓绒 |

### 2.2 基于场景的推荐算法

#### 2.2.1 算法原理

场景推荐算法根据用户设置的场景偏好（如通勤、约会、休闲、运动等），推荐适合特定场景的衣物。

#### 2.2.2 核心逻辑

```
输入：用户ID, 指定场景（可选）
输出：推荐衣物列表

流程：
1. 获取用户场景偏好（从 wardrobe_user_scene 表）
2. 如果指定场景，则仅使用指定场景
3. 根据场景偏好的频率权重排序
4. 查询衣物表中 occasion 字段匹配的衣物
5. 按收藏数和穿着次数排序
6. 返回Top 10结果
```

#### 2.2.3 场景偏好数据结构

```sql
-- 用户场景偏好表
wardrobe_user_scene (
    user_id,      -- 用户ID
    scene_code,   -- 场景编码（commute/date/casual/sport等）
    scene_name,   -- 场景名称
    frequency     -- 使用频率（1-5），作为权重
)
```

#### 2.2.4 SQL查询逻辑

```sql
SELECT c.id
FROM wardrobe_clothes c
WHERE c.del_flag = '0' 
  AND c.is_public = '1'
  AND (
    c.occasion LIKE '%commute%'    -- 通勤
    OR c.occasion LIKE '%date%'    -- 约会
    OR c.occasion LIKE '%casual%'  -- 休闲
  )
  AND c.id NOT IN (用户已有/已收藏的衣物)
ORDER BY c.favorite_count DESC, c.wear_count DESC
LIMIT 10
```

### 2.3 基于风格的推荐算法

#### 2.3.1 算法原理

风格推荐算法根据用户设置的风格偏好（如简约、韩系、复古、街头等），推荐符合用户审美的衣物。

#### 2.3.2 核心逻辑

```
输入：用户ID, 指定风格（可选）
输出：推荐衣物列表

流程：
1. 获取用户风格偏好（从 wardrobe_user_style 表）
2. 如果指定风格，则仅使用指定风格
3. 根据风格偏好的偏好程度排序
4. 查询衣物表中 style 字段匹配的衣物
5. 支持多风格匹配（衣物可能有多个风格标签）
6. 按收藏数排序，返回Top 10
```

#### 2.3.3 风格偏好数据结构

```sql
-- 用户风格偏好表
wardrobe_user_style (
    user_id,          -- 用户ID
    style_code,       -- 风格编码（minimalist/korean/vintage等）

```

#### 2.3.4 多风格匹配

衣物的 `style` 字段支持多个风格，以逗号分隔：

```

```

查询时使用 LIKE 进行模糊匹配：

```sql
WHERE c.style LIKE '%minimalist%' OR c.style LIKE '%korean%'
```

### 2.4 基于季节的推荐算法

#### 2.4.1 算法原理

季节推荐算法根据当前日期自动判断季节，推荐当季适穿的衣物。

#### 2.4.2 季节判断规则

| 月份 | 季节 |
|-----|------|
| 3-5月 | 春季 (spring) |
| 6-8月 | 夏季 (summer) |
| 9-11月 | 秋季 (autumn) |
| 12-2月 | 冬季 (winter) |

#### 2.4.3 核心逻辑

```java
private List<String> getCurrentSeasons() {
    int month = Calendar.getInstance().get(Calendar.MONTH) + 1;
    List<String> seasons = new ArrayList<>();
    
    if (month >= 3 && month <= 5) {
        seasons.add("spring");
    } else if (month >= 6 && month <= 8) {
        seasons.add("summer");
    } else if (month >= 9 && month <= 11) {
        seasons.add("autumn");
    } else {
        seasons.add("winter");
    }
    // 四季通用的衣物始终包含
    seasons.add("all_season");
    return seasons;
}
```

#### 2.4.4 SQL查询

```sql
SELECT c.id
FROM wardrobe_clothes c
WHERE c.del_flag = '0' 
  AND c.is_public = '1'
  AND (c.season IN ('spring', 'all_season'))
  AND c.id NOT IN (排除列表)
ORDER BY c.favorite_count DESC, c.wear_count DESC
LIMIT 10
```

### 2.5 协同过滤推荐算法

#### 2.5.1 算法原理

协同过滤（Collaborative Filtering）是推荐系统中最经典的算法之一。本系统采用**基于用户的协同过滤（User-Based CF）**，核心思想是：

> 如果用户A和用户B在过去有相似的行为（如收藏了相同的衣物），那么A喜欢的衣物，B可能也会喜欢。

#### 2.5.2 算法流程

```
输入：目标用户ID
输出：推荐衣物列表

流程：
1. 构建用户-物品评分矩阵
2. 计算用户相似度矩阵
3. 找到与目标用户最相似的K个用户
4. 获取相似用户喜欢但目标用户未接触的物品
5. 计算推荐分数并排序
6. 返回Top N推荐结果
```

#### 2.5.3 用户-物品评分矩阵

基于用户收藏行为构建评分矩阵：

```
           衣物1  衣物2  衣物3  衣物4  衣物5
用户A        1      1      0      1      0
用户B        1      0      1      1      0
用户C        0      1      1      0      1
用户D        1      1      0      0      1
```

- 1 表示用户收藏了该衣物
- 0 表示用户未收藏该衣物

#### 2.5.4 用户相似度计算

采用**余弦相似度（Cosine Similarity）**计算用户之间的相似程度：

$$
sim(u, v) = \frac{\sum_{i \in I_{uv}} r_{ui} \cdot r_{vi}}{\sqrt{\sum_{i \in I_u} r_{ui}^2} \cdot \sqrt{\sum_{i \in I_v} r_{vi}^2}}
$$

其中：
- $I_{uv}$ 是用户u和v共同评分的物品集合
- $r_{ui}$ 是用户u对物品i的评分

#### 2.5.5 Java实现

```java
private double calculateCosineSimilarity(Long user1, Long user2) {
    Map<Long, Double> ratings1 = userRatings.get(user1);
    Map<Long, Double> ratings2 = userRatings.get(user2);

    // 找到共同评分的物品
    Set<Long> commonItems = new HashSet<>(ratings1.keySet());
    commonItems.retainAll(ratings2.keySet());

    if (commonItems.isEmpty()) {
        return 0.0;
    }

    double dotProduct = 0.0;
    double norm1 = 0.0;
    double norm2 = 0.0;

    for (Long item : commonItems) {
        double r1 = ratings1.get(item);
        double r2 = ratings2.get(item);
        dotProduct += r1 * r2;
    }

    for (Double rating : ratings1.values()) {
        norm1 += rating * rating;
    }
    for (Double rating : ratings2.values()) {
        norm2 += rating * rating;
    }

    // 增加共同物品数量的惩罚因子
    double penalty = Math.min(1.0, commonItems.size() / 5.0);
    return (dotProduct / (Math.sqrt(norm1) * Math.sqrt(norm2))) * penalty;
}
```

#### 2.5.6 推荐分数计算

对于目标用户u未接触的物品i，推荐分数计算公式：

$$
score(u, i) = \frac{\sum_{v \in S_u} sim(u, v) \cdot r_{vi}}{\sum_{v \in S_u} sim(u, v)}
$$

其中：
- $S_u$ 是与用户u相似的用户集合
- $sim(u, v)$ 是用户u和v的相似度
- $r_{vi}$ 是用户v对物品i的评分

#### 2.5.7 算法优化

1. **相似度阈值过滤**：只考虑相似度 > 0.05 的用户
2. **相似用户数量限制**：最多考虑30个最相似用户
3. **时间衰减因子**：近期行为权重更高
4. **共同物品惩罚**：避免偶然相似导致的过拟合
5. **冷启动处理**：新用户返回热门推荐

```java
// 时间衰减因子计算
private double getTimeDecayWeight(Long userId, Long itemId) {
    long actionTime = userActionTimes.get(userId).get(itemId);
    long currentTime = System.currentTimeMillis();
    long daysDiff = (currentTime - actionTime) / (1000 * 60 * 60 * 24);
    
    // 每天衰减1%
    return Math.pow(0.99, daysDiff);
}
```

### 2.6 混合推荐算法

#### 2.6.1 算法原理

混合推荐（Hybrid Recommendation）融合多种推荐策略的结果，取长补短，提高推荐的准确性和多样性。

#### 2.6.2 融合策略

采用**加权融合（Weighted Hybrid）**策略：

$$
score_{final}(i) = \sum_{k=1}^{K} w_k \cdot score_k(i)
$$

其中：
- $K$ 是推荐方法的数量
- $w_k$ 是第k种方法的权重
- $score_k(i)$ 是第k种方法对物品i的推荐分数

#### 2.6.3 默认权重分配

| 推荐方法 | 权重 | 说明 |
|---------|------|------|
| 天气推荐 | 0.20 | 实时性强 |
| 场景推荐 | 0.20 | 个性化程度高 |
| 风格推荐 | 0.20 | 个性化程度高 |
| 季节推荐 | 0.15 | 稳定性好 |
| 协同过滤 | 0.25 | 社会化推荐 |

#### 2.6.4 核心实现流程

```
输入：用户ID, 城市名称（可选）
输出：融合后的推荐衣物列表

流程：
1. 并行调用5种推荐算法
2. 收集各算法的推荐结果
3. 归一化权重（仅考虑有结果的算法）
4. 对每个物品计算加权分数
5. 多来源加成（出现在多个推荐中的物品额外加分）
6. 排序并返回Top 10
```

#### 2.6.5 多来源加成

如果一个物品同时被多种推荐算法推荐，说明其匹配度更高，给予额外加成：

```java
// 多来源加成：每多一个来源加10%的分数
if (item.getSources().size() > 1) {
    double diversityBonus = 1.0 + (item.getSources().size() - 1) * 0.1;
    item.setScore(item.getScore() * diversityBonus);
}
```

#### 2.6.6 基于排名的分数计算

每种算法返回的是排序后的列表，使用排名计算分数：

```java
// 排名越靠前，分数越高
double rankScore = (double) (total - rank) / total;
double weightedScore = rankScore * weight;
```

例如，一个算法返回10个结果：
- 第1名：分数 = 10/10 = 1.0
- 第5名：分数 = 6/10 = 0.6
- 第10名：分数 = 1/10 = 0.1

## 四、冷启动问题处理

### 4.1 新用户冷启动

当新用户没有任何行为数据时：

1. **风格/场景推荐**：引导用户完成问卷，设置偏好
2. **协同过滤**：返回热门推荐（按收藏数+穿着次数排序）
3. **天气/季节推荐**：不依赖用户历史，可正常工作

### 4.2 新物品冷启动

当新上传的衣物没有被任何用户收藏时：

1. 依赖基于内容的推荐（风格、场景、季节匹配）
2. 在热门推荐中给予新物品曝光机会

## 五、性能优化

### 5.1 相似度矩阵预计算

协同过滤算法在初始化时预计算用户相似度矩阵，避免每次请求重复计算：

```java
public UserBasedCollaborativeFiltering(Map<Long, Map<Long, Double>> userRatings) {

```

### 5.2 推荐数量限制

- 每种算法最多返回10个结果
- 相似用户最多考虑30个
- 避免大数据量计算

### 5.3 排除列表优化

使用SQL的 `NOT IN` 子句排除用户已有和已收藏的衣物：

```sql

```
